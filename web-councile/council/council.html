    Pinned sessions

    Paste the exact conversation URL you want a council to always use for each service. If a tab with that conversation is already open, it's reused as-is; otherwise a new tab is opened straight to that link. Leave a field blank to fall back to "whichever tab is open." Auto-filled automatically once a fresh council's first prompt is sent — you don't need to fill these in by hand.

    Preferred model only applies when a brand-new chat is opened for that seat (an existing/reused conversation already has its model locked in). Leave blank to automatically pick the most capable/advanced option available; type "default" to leave the site's own default alone; or type part of a model's name (e.g. "Opus", "Pro", "Thinking") to match it.

      Consolidation judge

      Consolidation runs in its own dedicated session, separate from the members above, so it never appends a "merge these answers" turn into one of your real conversations. Pick which service judges, and optionally pin its own conversation URL and preferred model the same way.

      CRM connection

      Answers are POSTed as JSON to this endpoint. Point it at your CRM's inbound webhook (HubSpot workflow webhook, Salesforce Flow HTTP action, Zapier/Make/n8n catch hook, or an internal API).
